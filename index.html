My Malloc

1. This is a implementation of C's malloc/free with simplified mechanics.

2. Actual memory allocation is done by "sbrk" system call. The memory(free-memory) is managed with a self-define node structure with a doubly-linked list.

3. There is a merge functionality where freed memory node with adjacent address are merge back into a large node to avoid fragmentation.

4. Malloc & Free is thread safe with a no-lock mechanism, implemented with Thread Local Storage.

5. To further understand the topic, I studied the source code of glibc's implementation of malloc/free, realizing they have more layers of indirection, namely Arenas, Bins and Chunks to act like multiple level of caches in order to speed up the whole process.

Forward Proxy

1. This is a Forward Proxy written in C++ that support GET, POST and CONNECT method with a simple LRU to provide Caching ability when needed.

2. Proxy's connection with browser and destination server is done with TCP socket, using socket library in C.

3. The Proxy interpret the HTTP header to act accordingly. Specifically, alter the header when GET & POST to make the request instead of the browser and forward the result back from the server, cache the result when GET and finally build a tunnel when CONNECT so browser and destination server communicate directly

4. Each request from the browser is handled by a quest. A thread-pool is implemented to manage the thread resources. Epoll is used as a IO multiplexing mechanism to handle large amount of request.

Mini-UPS

1. This is a Full-stack project where we built a UPS-like website that can work with other Amazon-like website to mimic a full cycle of online shopping & delivering.

2. The website is built with Python using Django framework. The front-end webpage and back-end database(postgresql) scheme is built with Django and the back-end communication and parsing is done in a seperate component decoupled from Djnago framework.

3. The front-end allow viewing of package status and altering of package destination(if the package is not yet out for delivery). The back-end communicate with mini-Amazon under a google-protobuf API set by multiple teams as the communication protocol.

4. Due to the good design and well-defined protocol, different server can build their server in whichever language, in our case, python or Java, we still have a very good compatibility.
